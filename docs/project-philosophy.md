# Project Philosophy

## Purpose

This project is not simply a website.

It is a reusable framework for building high-performing local service business websites that combine excellent SEO, strong user experience, and maintainable architecture.

Every improvement made to this project should benefit future websites built from the framework.

The goal is to continuously improve the framework instead of solving the same problem repeatedly.

---

# Core Principles

## Build once. Reuse everywhere.

Whenever possible, create configurable systems instead of one-off solutions.

If a feature is useful for one business, consider whether it should become part of the framework so every future website benefits.

Avoid hardcoded business-specific logic when a configurable approach is possible.

---

## People first. Search engines second.

Every page should be written for homeowners first.

SEO exists to help people discover the content, not to control how the content is written.

Helpful content will always outperform keyword stuffing over the long term.

---

## Content must provide value.

Every page should answer real questions.

Every paragraph should educate, help, or move the visitor toward taking action.

Remove unnecessary words.

Avoid filler.

Avoid generic marketing language.

---

## Simplicity wins.

Prefer simple solutions over clever ones.

If a feature requires unnecessary complexity, rethink the approach.

The framework should be understandable by another developer without lengthy explanations.

---

## Data drives the website.

Business information should live in configuration and data files whenever practical.

Pages should be generated from structured data instead of duplicated templates.

Adding a new service or city should require changing data, not rewriting code.

---

## SEO is part of the architecture.

SEO is not an afterthought.

The framework should naturally produce:

* logical URLs
* strong internal linking
* useful page hierarchy
* structured data
* XML sitemaps
* robots.txt
* canonical URLs
* crawlable navigation

without requiring custom work for every website.

---

## Performance matters.

Fast websites provide a better user experience.

Avoid unnecessary JavaScript.

Avoid unnecessary dependencies.

Keep pages lightweight whenever possible.

Every feature should justify its performance cost.

---

## Build for maintenance.

Future developers should be able to understand the project quickly.

Clear organization is more valuable than clever implementation.

Document important decisions.

Keep business-specific logic separated from framework logic.

---

## Local SEO philosophy

Every business has one or more primary office locations.

Primary office cities should be represented by the homepage and core service pages.

Surrounding markets should be represented by service-area pages.

Avoid creating duplicate pages targeting the same search intent.

Internal linking should reinforce the relationship between the primary market and surrounding service areas.

---

## AI is a development tool.

AI should improve quality and productivity.

It should never reduce quality.

All AI-generated work should follow the project's documented standards.

Documentation is the source of truth.

Prompts should reference documentation rather than replacing it.

---

# Continuous Improvement

Every completed project should improve this framework.

When a better architecture, pattern, or process is discovered, evaluate whether it belongs in the framework so future websites benefit.

The framework should become more capable over time while remaining simple to understand and maintain.
